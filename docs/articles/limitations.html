    List of known limitations

    This is a working list of limitations of capability that may be addressed in further work

    Plant Simulator

    • PlantSimulator initalizes systems only to steady-state, it is not possible to start in a transient state
    • The code is not tested for a wide variety of complex configurations, so it is likely that the simulator may fail to initialize in some complex scenarios, as of yet undiscovered.

    Unit Simulator

    • UnitSimulator does not support advanced control involoving more than one PID-controller like cascade, select control/tracking or split range (this can be simualted using PlantSimulator)

    PID-Controller

    • the tracking functionality of the included PID-controller needs work(reformulation?), the example of tracking is unfinished as a result
    • split range has not been verified in a test
    • the uncertainty of estimated PID-paramters is currently not calculated or stored, but it is possible to do so

    Disturbance estimaton

    • a disturbance signal is always assumed to be zero at the start of the dataset
    • the disturbance estimation makes assumptions about u[0] being at steady-state and formulates model related to this u0=u[0]
    • the current closed-loop unit identifier cannot find dynamic models, for dynamic models some of the transients thus bleed into the estimated disturbance
    • disturbance estimation is only availabe currently for systems that the UnitSimulator can simulate, so not for advanced control schemes.
